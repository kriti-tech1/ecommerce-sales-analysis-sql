# ecommerce-sales-analysis-sql

   # E-Commerce Sales Analysis using PostgreSQL 

## Project Overview

This project analyzes e-commerce sales data using PostgreSQL to understand customer purchasing behavior, product performance, revenue generation, and sales trends. The project includes database design, table relationships, advanced SQL queries, and business-focused data analysis.

---

## Database Schema

The database contains the following tables:

### Customers Table

Stores customer information:

* Customer ID
* Customer Name
* City

### Products Table

Stores product-related details:

* Product ID
* Product Name
* Category
* Product Price

### Orders Table

Stores customer order information:

* Order ID
* Customer ID
* Order Date

### Order Details Table

Stores product purchase details:

* Order Detail ID
* Order ID
* Product ID
* Quantity

---

## Entity Relationship Flow

customers → orders → order_details → products

## Business Problems Solved

### Revenue Analysis

* Total revenue generated by each product
* Revenue generated by each product category
* Running total revenue over time

### Customer Analysis

* Highest revenue-generating customer
* Customers placing multiple orders
* Customers spending above average
* Customers purchasing across multiple categories
* Customers with high average order value

### Product Analysis

* Highest revenue-generating product by category
* Products never ordered
* Most expensive product purchased by each customer

### Order Analysis

* First and latest order date of customers
* Customers placing orders after their first purchase

### Ranking & Window Function Analysis

* Category ranking based on revenue
* Revenue comparison using DENSE_RANK()
* Running revenue calculations using window functions
